2. CSS Positioning :  

 a. Relative : 
    1. the element is positioned relative to its previous position 
    2. Gap is maintained when it is positioned 

 b. Absolute : 
    1. the element is positioned absolute to the closest positioned parent 
    2. Gap is filled 

 c. Static : (dafault) (cannot apply properties like top, left, bottom , right , z-index etc)

 d. Fixed : 
   1. the element is positioned according to browser window 
   2. it is fixed at the position throughout the web page 

 e. Sticky : 
   1. the element is positioned but when the parent component is disappeared from the window (like in scroll) then is also gone with that 

   2. there are 3 consitions without which it does not work : 
     - the containe should be scrollable 
     - top, bottom , etc should be there without this no sicky 
     - overflow property of parent container should be visible not hidden 


3. CSS Size Unit : 
   
 a. px : it is 1/96 the of the inch 

 b. em : 
    1. in this padding and margin remains unchanged 
    2. fontsize is calulated acc to parent i.e 1 em = fontsize of parent 
    3. width is calulated according to font size of it's own 

 c. rem : simply is nothing just according to root element font size i.e 1 rem = 16px

 d. vh : height equals to window height 

 e. vw : width equals to window width 

 f. % : it is given according to the parent component

5. Flexbox :
 taking analogy that we have a big conatainer in that we have 4 boxes (divs)

  - now we have to make the container , display :flex , using this the container becomes a flexbox and the items in that are k/a flex items 
  
  - Flex-wrap : In this the items which are in the container are wrapped when we change the width of the container 

  - Flex-direction : 
    a. row : in this the items are aligned in the row wise fashion (in this we have hor. axis)
    b. column : in this the items are aligned in the column wise fashion (in this we have vertical axis)
    c. row-reverse
    d. column-reverse   

  - Justify-content : (according to the axis - depends on flex-direction)
    a. start : align items at the start of the axis 
    b. end : at end
    c. center : at center 
    d. space-between : 1st and last item we be placed at ends and other are filled evenly 
    e. space-around : as name suggests each item will be having space in right and left part and all will be having even space 
    f. space-evenly : every box will be at equal distant from each other and with axis end also 

  - Align-items : (items are aligned according to vertical axis always which is placed at exatcly middle of the screen) 
    a. flex-start 
    b. flex-end 
    c. center  
    d. stretch (default)

  - Now understanding properties of flex item : 
   
  - order : it is a number value lower to higher arranges the flex items in start to end position 

  - flex-shrink : if items are shrinking then it is like multiplier if it is 2 for an item then that means that the box will be shrinking at speed of  2 times according to others 

  - flex-grow : same but for expansion factor simple 

  - flex-basis : it is used to give dimension of the item i.e width 

  - align-self : (according to cross axis just like align items axis)
    a. stretch 
    b. center 
    c. end 
    d. start

  16. Optimisation topic in CSS : 
    
     so here is how pipeline works : 
      
      1. HTML - DOM 
      2. CSS - CSSOM 
      3. Render Tree - It is combination of DOM and CSSOM, in this only nodes are present which are to be visible or shown on the screen 
        eg : <body>
              <h1>Hello</h1>
              <p style="display:none">Hidden</p>
              <span>World</span>
             </body>

             body
            ├── h1 ("Hello")
            └── span ("World")

       4. Layout (Reflow) : In this width , height, position etc are added to the nodes to create the layout 
       5. Paint : In this color , border etc are applied to the nodes 
       6. Composite : In this the layer is moved to GPU for optimization ,no reflow or repaint occurs here , that's why transform is faster and mor optimized than positioning
