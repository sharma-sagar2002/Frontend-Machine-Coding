1. Explain the async behaviour and event loop in JS ? 

A . js is a single threaded language which means it has a call stack and one by one function execution takes place and js does not waits for anything which means it is synchronous . Now timers , fetch , event listeners these all are given by web browser not by js , and to support async tasks we have callback queue , so in this callback method is pushed (eg : in timer when time is completed then callback is pushed to callback queue) and there comes event loop which continuously monitors the call stack when it is empty and the function is ready to get executed it pushes the callback in the callstack . Now for those tasks in which promises comes i.e fetch , in that the function is pushed into microtask queue , also the priority of execution of microtask queue is higher than callback queue . So , promise callbacks are executed first and when there is no fetch method left in microtask queue then callback queue methods got pushed into callstack

3. How can we achieve paralled execution in promises ?

a. Promise.all() 
   in this it waits for all promises to get resolved but if any of the promises is rejected then it gives the error as output which that rejected promises is giving , else it gives the array of resolved values 

   eg : promise.all([p1,p2,p3]).then ((res)=>{
    console.loop(res);
   }).catch ((e)=> console.log(e))

b. Promise.allSetteled() 
   in this it waits for all promises to get settled whether resolved or rejected and gives the array of result as output 

c. Promise.race() 
   return a value bcz it waits for first one to get settled either resolved or rejected 

d. Promise.any() 
   it is success seeking , bcz it waits for first one promise which is get resolved and return the value of that but it no one got resolved then it gives an aggregate error in which array of all promises errors is there
